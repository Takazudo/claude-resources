# doc/ — Claude Resources Documentation Site

Astro-based static site that renders Claude Code resources (CLAUDE.md, commands, skills, agents) as browsable documentation.

## Quick Start

```bash
pnpm i
pnpm dev          # astro dev on port 4891 (HMR, for development)
pnpm dev:stable   # build-then-serve on port 4892 (used by Tauri app)
pnpm build        # production build to dist/
```

## Architecture

- **Astro 5** static site with MDX, React, Tailwind CSS
- **`src/integrations/claude-resources/`** — reads `~/.claude/` at build time, generates MDX pages for CLAUDE.md, commands, skills, agents
- **`scripts/dev-stable.js`** — build-then-serve mode with SSE live-reload (the Tauri app uses this)
- **`scripts/watch-dirs.js`** — configures which directories trigger rebuilds (src/, ~/.claude/CLAUDE.md, commands/, skills/, agents/)
- **`src/config/settings.ts`** — site configuration (color scheme, navigation, integrations)

## Two Dev Modes

| Mode | Port | Command | Use Case |
|------|------|---------|----------|
| `dev` | 4891 | `astro dev` | Development with HMR |
| `dev:stable` | 4892 | `node scripts/dev-stable.js` | Tauri app sidecar (full build + SSE reload) |

## Key Directories

- `src/content/docs/` — static doc pages (hand-authored)
- `src/content/docs/claude*/` — **generated** by claude-resources integration (do not edit)
- `src/integrations/` — Astro integrations (claude-resources, search-index, doc-history, etc.)
- `src/components/` — React/Astro components
- `scripts/` — dev-stable.js, watch-dirs.js
- `scripts/__tests__/` — tests (run with `node --test scripts/__tests__/*.test.js`)

## Live-Reload (dev:stable)

`dev-stable.js` provides SSE-based live-reload:
1. File change detected in watched dirs → `building` SSE event → spinner overlay appears
2. Astro build completes → dist swapped → `rebuild` SSE event → browser reloads

Watcher ignores `content/docs/claude*` (generated by builds) to prevent infinite rebuild loops.

## Tests

```bash
node --test scripts/__tests__/watch-dirs.test.js        # unit: watch directory config
node --test scripts/__tests__/watch-integration.test.js  # integration: watcher triggers
node --test scripts/__tests__/live-reload.test.js        # e2e: SSE, spinner, content update
```

Tests use Node.js built-in test runner. Integration/e2e tests start the actual dev-stable server and modify real files (with cleanup in `finally` blocks).
