# ASCII Circuit Diagram Golden Rules

These rules ensure clear, unambiguous ASCII circuit diagrams.

## Rule 1: Never Cross Lines Without Junctions

**вқҢ WRONG** - Lines crossing without junction (ambiguous):
```
Signal A  в”Җв”Җв”јв”Җв”Җ  (is this connected or just passing?)
            в”Ӯ
         Signal B
```

**вң… CORRECT** - Route around to avoid crossing:
```
Signal A  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ  (clearly not connected)

         Signal B
            в”Ӯ
```

**вң… CORRECT** - Use explicit junction when signals DO connect:
```
Signal A  в”Җв”Җв”¬в”Җв”Җ  (T-junction: A and B connect here)
            в”Ӯ
         Signal B
```

## Rule 2: Never Cross Lines Over Text Labels

When a vertical or horizontal line passes over a text label, it creates ambiguity.

**вқҢ WRONG** - Lines crossing over labels:
```
Output в”Җв”Җв”¬в”Җв”ҖвҶ’ Load
         в”Ӯ
        GND      вҶҗ Label
         в”Ӯ       вҶҗ Is this line connected to GND label?
        C1
```

**вң… CORRECT** - Remove intermediate labels:
```
Output в”Җв”Җв”¬в”Җв”ҖвҶ’ Load
         в”Ӯ
         в”ңв”ҖвҶ’ C1 в”ҖвҶ’ GND
            470ВөF
```

**Solutions:**
1. Route lines around labels
2. Remove intermediate labels that would be crossed
3. **Route vertically in opposite direction** - If downward line crosses labels, route upward

## Rule 3: Use Label Notation for Distant Connections

When connections would require crossing wires, use arrow-to-label notation:

**вң… CORRECT**:
```
IC Pin в”ң2в”ҖвҶ’ Tap      вҶҗ Pin points to "Tap" label (no wire drawn)

Output в”Җв”¬в”ҖвҶ’ R1 в”Җв”Җв”¬в”ҖвҶ’ Tap в”ҖвҶ’ R2 в”ҖвҶ’ GND
        в”Ӯ         вҶ‘
        в”Ӯ    This "Tap" matches the label above
```

Examples:
- `FB в”ң2в”ҖвҶ’ Tap` means "FB pin 2 connects to the point labeled 'Tap'"
- `GND в”ң1в”ҖвҶ’ GND` means "GND pin 1 connects to system ground"

### Use "GND" Label Often (Important Practice)

**Best Practice**: In circuit design, use "GND" as a label frequently instead of drawing every ground connection.

**Why**:
- GND connections are extremely common in circuits
- Trying to express every GND connection accurately makes diagrams messy and cluttered
- Using `вҶ’ GND` notation keeps diagrams clean and readable

- Diode anodes (in flyback positions) вҶ’ `в”ҖвҶ’ GND`
- Any component that needs ground connection

**Exception**: When showing a specific ground topology is important (e.g., star ground, ground loops), draw the actual connections. For most circuit diagrams, labels are clearer.

## Rule 4: Show Parallel Components as Vertical Drops

Components connected in parallel (between signal and GND) should be drawn vertically:

**вң… CORRECT** - Single capacitor:
```
Output в”Җв”Җв”¬в”Җв”ҖвҶ’ Load
         в”Ӯ
        C1 (470ВөF)   вҶҗ Clearly parallel
         в”Ӯ
        GND
```

**вң… BEST PRACTICE** - Multiple parallel capacitors (filter pairs):
```
Power в”Җв”Җв”¬в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”¬в”Җв”Җв”Җ To IC
        в”Ӯ                в”Ӯ
       C1               C2
      100nF            470ВөF
     ceramic        electrolytic
     (CLOSE!)        (farther)
        в”Ӯ                в”Ӯ
       GND              GND
```
Key points:
- T-junction (в”¬) at TOP on the power rail
- Capacitor labels in the middle of vertical lines
- Clean vertical drops to GND (no extra junctions at bottom)
- **Left-to-right order**: Ceramic (CLOSE to IC) on LEFT, electrolytic (farther) on RIGHT
- Can show physical placement hints (ceramic close to IC, electrolytic farther)
- Common in power supply designs (bulk + decoupling capacitor pairs)

**вқҢ WRONG** - Looks like series:
```
Output в”Җв”Җ C1 в”Җв”Җ Load   вҶҗ Confusing! Looks like C1 blocks current
```

**вқҢ WRONG** - Junction at bottom instead of top:
```
        C1               C2
         в”Ӯ                в”Ӯ
Power в”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җ
         в”Ӯ                в”Ӯ
        GND              GND
```
This makes it look like the junction is at GND, not at the power rail!

## Box-Drawing Character Reference

### Basic Junctions (Most Common)

| Symbol | Code | Use Case |
|--------|------|----------|
| `в”ң` | 251C | T-junction right - signal branches to the right |
| `в”Ө` | 2524 | T-junction left - signal branches to the left |
| `в”¬` | 252C | T-junction down - signal splits downward |
| `в”ҙ` | 2534 | T-junction up - signal merges from below |

### Lines

| Symbol | Code | Use Case |
|--------|------|----------|
| `в”Җ` | 2500 | Horizontal trace (standard signals) |
| `в”Ғ` | 2501 | Heavy horizontal (power rails, main paths) |
| `в”Ӯ` | 2502 | Vertical trace (standard signals) |
| `в”ғ` | 2503 | Heavy vertical (power rails, main paths) |
| `в•җ` | 2550 | Double horizontal (differential pairs, parallel conductors) |
| `в•‘` | 2551 | Double vertical (differential pairs, parallel conductors) |

### Corners

| Symbol | Code | Use Case |
|--------|------|----------|
| `в”Ң` | 250C | Corner down-right |
| `в”җ` | 2510 | Corner down-left |
| `в””` | 2514 | Corner up-right |
| `в”ҳ` | 2518 | Corner up-left |
| `в”Ҹ` | 250F | Heavy corner down-right (power) |
| `в”“` | 2513 | Heavy corner down-left (power) |
| `в”—` | 2517 | Heavy corner up-right (power) |
| `в”ӣ` | 251B | Heavy corner up-left (power) |

### Cross Junctions

| Symbol | Code | Use Case |
|--------|------|----------|
| `в”ј` | 253C | **AVOID** - Ambiguous 4-way cross |
| `в•Ӣ` | 254B | Heavy cross - only if truly 4-way connection |

**Note**: Avoid `в”ј` and `в•Ӣ` unless all 4 directions genuinely connect. Use T-junctions instead for clarity.

## Rule 6: Use Unicode Characters for Clarity

**Why Unicode is BETTER than ASCII**:
- **Box-drawing**: `в”ң в”Ө в”¬ в”ҙ в”Җ в”Ӯ в”Ң в”җ в”” в”ҳ` show exact junction direction - unambiguous
- **Arrows**: `вҶ’ вҶҗ` are standard directional symbols
- **ASCII `+`** is ambiguous - could connect in any direction
- Clear visual topology prevents false junction errors
- Easier to trace signal paths in preview

**Best practice: Use full Unicode character set**

**вң… CORRECT** - Unicode throughout:
```
Output в”Җв”Җв”¬в”ҖвҶ’ Load
         в”Ӯ
        C1 (470ВөF)
         в”Ӯ
        GND
```
The `в”¬` clearly shows: connects down AND horizontally to Load
The `вҶ’` clearly shows signal direction

**вқҢ WRONG** - ASCII `+` creates ambiguous junctions:
```
Output --+-> Load
         |
        C1 (470uF)
         |
        GND
```
The `+` is ambiguous - does it connect left? Right? Up? Down? All directions?

**CRITICAL**: Using ASCII `+` for junctions was the cause of false junction errors where CC1/CC2 appeared connected to capacitor lines when they shouldn't be!

## Rule 7: Use Boxes to Identify Components

**Purpose**: Boxes visually identify individual components and group their labels/values together.

**Rule**: Draw a box around each component (IC, capacitor, resistor, connector, etc.)

**вң… CORRECT** - Each component has its own box with proper padding:
```
     в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Җв”Җв”Җв”Җв”Җв”Ө C1   в”ңв”Җв”Җв”Җв”Җв”Җ
     в”Ӯ 10ВөF в”Ӯ
     в””в”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```
The box makes it clear: "This is C1, value 10ВөF"

**вң… CORRECT** - Box has enough width for content with padding:
```
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ GND (B12,A12) в”ңв”Җв”Җв”Җв”Җв”Җв”Җ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

**вқҢ WRONG** - Floating text is ambiguous:
```
     C1
      в”Ӯ
в”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җ
    10ВөF
```
Which line is C1? What does 10ВөF refer to?

**вқҢ WRONG** - Box too narrow, text touches borders:
```
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”ӮGND (B12,A12)в”ңв”Җв”Җв”Җв”Җв”Җв”Җ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

**Key points**:
- **One box per component** - don't group multiple components
- **Box should have enough width** - Include at least 1 space padding on each side of the text inside the box
- Box contains: component designator (U1, C1, R1) and value/type
- For ICs: show pin numbers inside the box
- For connectors: show pin names inside the box
- Makes component identification immediate and unambiguous

**Problem**: When circuit details become complicated within a diagram, they create clutter and reduce clarity.

**Rule**: When a complicated layout occurs in a diagram, separate the layout into a new diagram block.

```

This is unclear - where does +5V come from? How is the LED connected?

**When to separate:**
- Circuit detail involves components from other diagrams (e.g., +5V from a different stage)

**Benefits:**
- Main diagram stays clean and focused on primary connections
- Detail diagram shows full circuit topology with all components
- Easier to understand both overview and implementation
- Can add notes explaining operation without cluttering main diagram

## Rule 9: Separate Functional Blocks for Complex Circuits

**Best Practice**: For complex circuits (buck converters, voltage inverters, multi-stage circuits), break the diagram into clearly labeled functional sections.

**Layout principles:**
1. **Branch right, then down** - Use `в””в”Җв”Җв”Җ` to create branches that extend right first, then go vertical
2. **Main flow horizontal, branches vertical** - Keep signal flow left-to-right, use vertical lines for branches to GND or parallel components
3. **One function per section** - IC, switching node, input caps, feedback network, etc.

**вң… CORRECT** - Separated sections with clear labels:
```
LM2596S-ADJ IC:
                    в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ

- **No line crossings** between different functional blocks
- **Crystal clear** component grouping
- **Easy to trace** signal paths within each section

- **Easy to modify** individual sections without affecting others

- Complex circuits with >5 components
- Circuits where components serve distinct functions
- Any time a single-diagram layout would require crossing lines over labels
- Buck/boost converters (IC + switching node + filters + feedback)
- Multi-stage circuits (inverter + regulator + protection)

## Rule 10: No Flying/Disconnected Lines

**Problem**: Vertical lines must maintain exact column alignment throughout their entire path. If the column position shifts even by one character, the line appears to "float" or disconnect.

**вқҢ WRONG** - Line appears disconnected (off by 1 space):
```
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ     в”Ӯ
                                                      в”Ӯ  вҶҗ Shifted right by 1
                                    в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
```

**вң… CORRECT** - Perfect column alignment:
```
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ     в”Ӯ
                                                     в”Ӯ  вҶҗ Same column position
                                    в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
```

**Critical**: These subtle misalignments are often invisible in plain text but become obvious in monospace preview. **Always preview and carefully inspect vertical line alignment before finalizing.**

**How to detect and fix**:
1. **Preview in monospace** - Alignment issues become visible
2. **Trace each vertical line** - Follow from start to end, checking every row
3. **Count character positions** - Verify exact column numbers if needed
4. **Look for "broken" appearance** - Lines that seem to disconnect or float
5. **Fix immediately** - Adjust spacing to restore perfect alignment

**Common causes**:
- Labels with different character widths pushing lines around
- Manual spacing adjustments that shift subsequent lines
- Copy-paste errors that don't preserve exact spacing
- Forgetting to count junction symbols as part of the vertical path

### Example: Box Connection Alignment

**Problem**: When connecting vertical lines to boxes, the junction must be IN the box border, not beside it.

**вқҢ WRONG** - Line beside box (misaligned):
```

The vertical lines appear disconnected from the box!

**вң… CORRECT** - Junction IN the box border:
```
     в”Ӯ
в”Ңв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”җ

в””в”Җв”Җв”Җв”Җв”¬в”Җв”Җв”Җв”ҳ

Use `в”ҙ` (top) and `в”¬` (bottom) to replace part of the border and show the connection point.

**Rule**: When a vertical line connects to a box's top or bottom:
- Replace `в”Җ` in the top border with `в”ҙ` (T-junction down)
- Replace `в”Җ` in the bottom border with `в”¬` (T-junction up)

**Note**: This is a common alignment issue that could be automatically detected and fixed by a linting script. The pattern is consistent: any vertical line above/below a box should have a junction symbol in the box border.

## Component Notation

- **ICs**: Use part numbers (CH224D, LM2596S, etc.)
- **Passives**: Show values with units (10ВөF, 5.1kО©, 33ВөH) OR use notation if values break alignment
- **Voltages**: Show at each stage (+15V, +13.5V, +12V)
- **Currents**: Show max ratings (1.2A, 800mA)

**Component Value Placement**:

When component values fit cleanly:
```
            |
           GND
            |
           C3
          470uF
            |
```

When values break column alignment, use text notation instead:
```
            |
           GND
            |
           C3
            |
```

Then list values after the diagram:
```
Component values:
  C3: 470uF
  R1: 10kО©
  L1: 100uH
```

**Guideline**: If adding a value label shifts the vertical line position, use text notation instead.

## Validation Checklist

Before finalizing any diagram:

1. вң… No line crossings without explicit junctions
2. вң… No lines crossing over text labels
3. вң… All components properly connected (no floating components)
4. вң… Parallel components shown as vertical drops to GND
5. вң… Diode polarity correct (cathode/anode orientation)
6. вң… ASCII-only characters used (no Unicode box-drawing or arrows)
7. вң… No flying/disconnected lines (vertical lines maintain column alignment)
8. вң… Consistent column alignment (no sliding vertical bars)
9. вң… Label notation used for distant connections
10. вң… Preview verified in monospace rendering
